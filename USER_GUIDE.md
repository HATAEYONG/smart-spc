## 🎯 시스템 개요

### 시스템 특징
- **AI 기반 품질 예측**: 머신러닝을 활용한 실시간 품질 이상 예측
- **통합 관제 시스템**: 품질, 설비, 치공구, 생산 현황을 한 곳에서 관리
- **SPC & 통계 분석**: 통계적 공정 관리 도구 모음
- **ERP/MES 연계**: 외부 시스템과의 데이터 연동 지원

### 기술 스택
- **Frontend**: React + TypeScript + Vite
- **UI Framework**: Tailwind CSS
- **차트**: Recharts
- **Backend**: Django + Django REST Framework (구현 예정)

### 3. 실시간 생산 모니터링

**위치**: 기본 기능 → 생산 모니터링

**주요 기능**:
- ✅ 생산 라인별 실시간 현황
- ✅ OEE(Overall Equipment Effectiveness) 모니터링
- ✅ 불량률 실시간 추적

**OEE 구성**:
- **가동률(Availability)**: 설비 가동 시간 비율
- **성능률(Performance)**: 실제 생산 속도 대비 이상적인 속도 비율
- **양품률(Quality)**: 양품 비율

```
OEE = 가동률 × 성능률 × 양품률
```

**OEE 등급**:
- 85% 이상: 세계급 (World Class)
- 60-85%: 보통
- 60% 미만: 개선 필요

## ❓ FAQ

### Q1: 새로운 품질 이슈를 등록하는 방법은?
**A**:
1. SPC & 분석 → 품질 이슈 추적 메뉴 이동
2. "새 이슈 등록" 버튼 클릭
3. 제목, 설명, 제품 정보, 중요도 입력
4. 저장 후 4M 분석 및 8단계 문제 해결 단계 입력

### Q2: 설비 건강 점수는 어떻게 계산되나요?
**A**:
- 최근 수리 이력, 다운타임, 연령 등을 종합하여 산출
- 100점: 최상, 0점: 최악
- 85점 미만: 주의 필요

### Q3: OEE 85% 달성을 위해서는?
**A**:
```
OEE = 가동률 × 성능률 × 양품률
모든 요소가 최소 95% 이상이어야 OEE 85% 달성
```

### Q4: 치공구 교체 시점을 어떻게 알 수 있나요?
**A**:
- 치공구 관리 → 예측 고도화 메뉴 확인
- "잔존 수명" 및 "교체 시급성" 참조
- "즉시" 표시되면 즉시 교체 필요

### Q5: 작업지시 생성 시 유의할 점은?
**A**:
- 설비와 치공구가 할당되어 있는지 확인
- 설비 건강 점수 85점 이상 권장
- 치공구 잔존 수명이 충분한지 확인
- 위험도 "높음" 미만일 때 작업 지시

### Q6: ERP 연결 시 필요한 정보는?
**A**:
- API Endpoint URL
- 인증 방식 (API Key, OAuth, Basic Auth)
- 인증 키/토큰
- 동기화 주기 (분 단위)
- 동기화할 데이터 유형

### Q7: 데이터 내보내기는 어떻게 하나요?
**A**: 각 페이지 우측 상단의 "내보내기" 버튼 클릭 (현재 UI만 준비됨, 실제 내보내기 기능은 백엔드 연동 시 구현)

## 🔧 Troubleshooting

### 문제 1: 메뉴가 보이지 않아요
**해결**:
- 왼쪽 메뉴 바를 스크롤 해보세요
- 화면을 새로고침 (F5)

### 문제 2: 데이터가 저장되지 않아요
**해결**:
- 필수 항목(*) 모두 입력했는지 확인
- 현재는 mock 데이터이므로 새로고침하면 초기화됨
- 실제 저장은 백엔드 연동 후 가능

### 문제 3: 연결 안됨 메시지가 떠요
**해결**:
- 이미 WebSocket 연결을 비활성화함
- 알림 기능은 백엔드 구현 후 활성화 예정

### 문제 4: 페이지가 느려요
**해결**:
- 브라우저 캐시 삭제
- 최신 브라우저 사용 권장
- Chrome, Edge 권장

### 문제 5: 빌드 에러가 발생해요
**해결**:
```bash
cd frontend
rm -rf node_modules
npm install
npm run build
```
